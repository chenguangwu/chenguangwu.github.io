三、登录mysql
初始状态下，管理员root，密码为空，默认只允许从本机登录localhost
设置密码
[root@yang ~]# mysqladmin -uroot password "123" 设置初始密码 由于原密码为空，因此-p可以不用
[root@yang ~]# mysqladmin -uroot -p123 password "456" 修改mysql密码

命令格式:
[root@yang ~]# mysql -h192.168.2.168 -uroot -p123
[root@yang ~]# mysql -uroot -p
[root@yang ~]# mysql 以root用户登录本机，密码为空


忘记密码：
方法一： 删除授权库mysql，重新初始化
[root@yang ~]# rm -rf /var/lib/mysql/mysql
[root@yang ~]# service mysqld restart
[root@yang ~]# mysql
警告：所有的授权信息将全部丢失!!!

方法二：启动时，跳过授权
[root@yang ~]# vim /etc/my.cnf //mysql主配置文件
[mysqld]
skip-grant-table
[root@yang ~]# service mysqld restart
[root@yang ~]# mysql

===整形
年龄，游戏level
分为有符号和无符号，默认有符号，使用约束条件unsigned存储正值
一般使用INT
不需要指定宽度

===浮点型
身高，体重，工资
FLOAT单精度
指定宽度 gz float(10,2) //总宽度10位，小数部分占2位

===字符串类型
姓名、家庭住址、爱好
char 定长 name char(10) 无论是否够10个字符，都会占用10个字符
varchar 变长 named varchar(10) 仅占用字符长度的空间
注：宽度，最大长度的限制
字符类型的值在插入需要使用引号

常见的是单选按扭和复选框
enum 单选 只能在给定的范围内选一个值，如性别 sex 男/女
set 多选 在给定的范围内可以选择一个或一个以上的值（爱好1,爱好2,爱好3...）

完整性约束
作用：用于保证数据的准备性和一致性
===================================================================
约束条件 说明
PRIMARY KEY (PK) 标识该字段为该表的主键，可以唯一的标识记录
FOREIGN KEY (FK) 标识该字段为该表的外键
NOT NULL 标识该字段不能为空
UNIQUE KEY (UK) 标识该字段的值是唯一的
AUTO_INCREMENT 标识该字段的值自动增长 （整数类型）
DEFAULT 为该字段设置默认值
===================================================================

unsigned


1. 是否允许为空，默认NULL，可设置NOT NULL，字段不允许为空，必须赋值
2. 字段是否有默认值，缺省的默认值是NULL，如果插入记录时不给字段赋值，此字段使用默认值
sex enum('boy','girl') not null default 'boy'
age int unsigned NOT NULL default 20 必须为正值（无符号） 不允许为空 默认是20
3. 是否是key
主键 primary key
外键 forengn key
索引 (index,unique...)
