配置DNS服务器

==正向区： 将域名解析为IP
==反向区： 将IP解析为域名
========================================================


一、主DNS服务器正向区
DNS软件包： bind bind-chroot
监听端口： 53/udp, 53/tcp

案例1：
1. 安装
[root@yangs ~]# yum -y install bind bind-chroot

2. 配置
正向记录
记录类型 ：
SOA 起始授权记录 第一条记录 规定主区域和辅助区域之间的复制各种条件
NS 名称服务记录 负责名称解析的区域
A 正向记录
CNAME 别名记录
PTR 反向记录
MX 邮件交换记录 邮件服务器
a. 主配置文件
[root@yangs ~]# mv /etc/named.conf /etc/named.conf.old 可选
[root@yangs ~]# vim /etc/named.conf
options {
directory "/var/named"; //数据库文件存放的位置
};

zone "uplooking.com" { //创建域uplooking.com
type master;
file "uplooking.com.zone";
};

记得将文件属组改为named
b. 数据库文件(区域文件)
[root@yangs ~]# vim /var/named/uplooking.com.zone
$TTL 7200
uplooking.com. IN SOA uplooking.com. root.uplooking.com. ( 2014012100 1H 15M 1W 1D)
uplooking.com. IN NS dns1.uplooking.com.
dns1.uplooking.com. IN A 192.168.10.66
www.uplooking.com. IN A 192.168.10.137
oa.uplooking.com. IN A 192.168.10.33

c. 解析测试

[root@tianyun ~]# chkconfig named on

二、主DNS服务器反向区
反向域名解析(也就是说 IP 地址到主机名的解析)通过 in-addr.arpa 域和 PTR 记录实现。
in-addr.arpa 域入口可以作成最不重要到最重要(least-to-most)顺序,从左至右阅读。这与 IP
地址的通常顺序相反。这样,一台 IP 地址为 10.1.2.3 的机器将会右对应的 in-addr.arpa 名称:
3.2.1.10.in-addr.arpa。
1. 反向区名是由网络地址决定的如：192.168.10.0/24的反向区名为10.168.192.in-addr.arpa
2. 反向区主机名和相应的正向区的主机名必须一一对应
3. 主机名必须全名
4. 反向区一般用于邮件服务器的反向解析
5. 反向区通常由ISP来实现

案例3：
为192.168.10.0/24网段建立反向区:
a. 主配置文件
[root@yangs ~]# vim /etc/named.conf
zone "10.168.192.in-addr.arpa" {
type master;
file "192.168.10.zone";
};

b. 数据库文件
[root@yangs ~]# cd /var/named/
[root@yangs named]# vim 192.168.10.zone
$TTL 3600

2014012200
1H
15M
1W
1D )
IN NS dns1.com.
66 IN PTR dns1.com.
16 IN PTR mail.tianyun.com.
20 IN PTR mail.uplooking.com.
