  1. 一、JSP EL语言定义  
  2.   
  3.    
  4.        EL 提供了在 JSP 脚本编制元素范围外使用运行时表达式的功能。脚本编制元素是指页面中能够用于在 JSP 文件中嵌入 Java 代码的元素。它们通常用于对象操作以及执行那些影响所生成内容的计算。JSP 2.0 将 EL 表达式添加为一种脚本编制元素。

  7. 二、JSP EL简介  
  8.   
  9. 1、语法结构  
  10.      ${expression}  
  11. 2、[ ]与.运算符  
  12.      EL 提供“.“和“[ ]“两种运算符来存取数据。  
  13.      当要存取的属性名称中包含一些特殊字符，如.或?等并非字母或数字的符号，就一定要使用“[ ]“。例如：  
  14.          ${user.My-Name}应当改为${user["My-Name"] }  
  15.      如果要动态取值时，就可以用“[ ]“来做，而“.“无法做到动态取值。例如：  
  16.          ${sessionScope.user[data]}中data 是一个变量  
  17. 3、变量  
  18.      EL存取变量数据的方法很简单，例如：${username}。它的意思是取出某一范围中名称为username的变量。  
  19.      因为我们并没有指定哪一个范围的username，所以它会依序从Page、Request、Session、Application范围查找。  
  20.      假如途中找到username，就直接回传，不再继续找下去，但是假如全部的范围都没有找到时，就回传null。  
  21.      属性范围在EL中的名称  
  22.          Page          PageScope  
  23.          Request          RequestScope  
  24.          Session          SessionScope  
  25.          Application      ApplicationScope

  87. 3、隐式对象  
  88.   
  89. JSP 表达式语言定义了一组隐式对象，其中许多对象在 JSP scriplet 和表达式中可用：  
  90.   
  91.   
  92.   
  93.   
  94.   
  95. pageContext  
  96.       
  97.   
  98. JSP 页的上下文。它可以用于访问 JSP 隐式对象，如请求、响应、会话、输出、servletContext 等。例如，${pageContext.response} 为页面的响应对象赋值。  
  99.   
  100.   
  101. 此外，还提供几个隐式对象，允许对以下对象进行简易访问：  
  102.   
  103.   
  104. 术语  定义  
  105.   
  106. param  
  107.       
  108.   
  109. 将请求参数名称映射到单个字符串参数值（通过调用 ServletRequest.getParameter (String name) 获得）。getParameter (String) 方法返回带有特定名称的参数。表达式 $(param.name) 相当于 request.getParameter (name)。  
  110.   
  111. paramValues  
  112.       
  113.   
  114. 将请求参数名称映射到一个数值数组（通过调用 ServletRequest.getParameter (String name) 获得）。它与 param 隐式对象非常类似，但它检索一个字符串数组而不是单个值。表达式 ${paramvalues.name) 相当于 request.getParamterValues(name)。  
  115.   
  116. header  
  117.       
  118.   
  119. 将请求头名称映射到单个字符串头值（通过调用 ServletRequest.getHeader(String name) 获得）。表达式 ${header.name} 相当于 request.getHeader(name)。  
  120.   
  121. headerValues  
  122.       
  123.   
  124. 将请求头名称映射到一个数值数组（通过调用 ServletRequest.getHeaders(String) 获得）。它与头隐式对象非常类似。表达式 ${headerValues.name} 相当于 request.getHeaderValues(name)。  
  125. cookie  将 cookie 名称映射到单个 cookie 对象。向服务器发出的客户端请求可以获得一个或多个 cookie。表达式 ${cookie.name.value} 返回带有特定名称的第一个 cookie 值。如果请求包含多个同名的 cookie，则应该使用 ${headerValues.name} 表达式。  
  126. initParam   将上下文初始化参数名称映射到单个值（通过调用 ServletContext.getInitparameter(String name) 获得）。  
  127.   
  128.   
  129. 除了上述两种类型的隐式对象之外，还有些对象允许访问多种范围的变量，如 Web 上下文、会话、请求、页面：  
  130.   
  131.   
  132. 术语  定义  
  133.   
  134. pageScope  
  135.       
  136.   
  137. 将页面范围的变量名称映射到其值。例如，EL 表达式可以使用 ${pageScope.objectName} 访问一个 JSP 中页面范围的对象，还可以使用 ${pageScope.objectName.attributeName} 访问对象的属性。  
  138.   
  139. requestScope  
  140.       
  141.   
  142. 将请求范围的变量名称映射到其值。该对象允许访问请求对象的属性。例如，EL 表达式可以使用 ${requestScope.objectName} 访问一个 JSP 请求范围的对象，还可以使用 ${requestScope.objectName.attributeName} 访问对象的属性。  
  143.   
  144. sessionScope  
  145.       
  146.   
  147. 将会话范围的变量名称映射到其值。该对象允许访问会话对象的属性。例如：  
  148.   
  149.   
  150. $sessionScope.name}   
  151.   
  152. applicationScope  
  153.       
  154.   
  155. 将应用程序范围的变量名称映射到其值。该隐式对象允许访问应用程序范围的对象。

  158. 三、特别强调：  
  159.   
  160. 1、注意当表达式根据名称引用这些对象之一时，返回的是相应的对象而不是相应的属性。例如：即使现有的 pageContext 属性包含某些其他值，${pageContext} 也返回 PageContext 对象。  
  161.   
  162. 2、 注意 <%@ page isELIgnored="true" %> 表示是否禁用EL语言,TRUE表示禁止.FALSE表示不禁止.JSP2.0中默认的启用EL语言。

  216. OUT:  
  217. <c:out value="/${logininfo.username}"/>  
  218. c:out>将value 中的内容输出到当前位置，这里也就是把logininfo 对象的  
  219. username属性值输出到页面当前位置。  
  220. ${……}是JSP2.0 中的Expression Language（EL）的语法。它定义了一个表达式，  
  221. 其中的表达式可以是一个常量（如上），也可以是一个具体的表达语句（如forEach循环体中  
  222. 的情况）。典型案例如下：  
  223. ? ${logininfo.username}  
  224. 这表明引用logininfo 对象的username 属性。我们可以通过“.”操作符引  
  225. 用对象的属性，也可以用“[]”引用对象属性，如${logininfo[username]}  
  226. 与${logininfo.username}达到了同样的效果。  
  227. “[]”引用方式的意义在于，如果属性名中出现了特殊字符，如“.”或者“-”，  
  228. 此时就必须使用“[]”获取属性值以避免语法上的冲突（系统开发时应尽量避免  
  229. 这一现象的出现）。  
  230. 与之等同的JSP Script大致如下：  
  231. LoginInfo logininfo =  
  232. (LoginInfo)session.getAttribute(“logininfo”);  
  233. String username = logininfo.getUsername();  
  234. 可以看到，EL大大节省了编码量。  
  235. 这里引出的另外一个问题就是，EL 将从哪里找到logininfo 对象，对于  
  236. ${logininfo.username}这样的表达式而言，首先会从当前页面中寻找之前是  
  237. 否定义了变量logininfo，如果没有找到则依次到Request、Session、  
  238. Application 范围内寻找，直到找到为止。如果直到最后依然没有找到匹配的  
  239. 变量，则返回null.  
  240. 如果我们需要指定变量的寻找范围，可以在EL表达式中指定搜寻范围：  
  241. ${pageScope.logininfo.username}  
  242. ${requestScope.logininfo.username}  
  243. ${sessionScope.logininfo.username}  
  244. ${applicationScope.logininfo.username}  
  245. 在Spring 中，所有逻辑处理单元返回的结果数据，都将作为Attribute 被放  
  246. 置到HttpServletRequest 对象中返回（具体实现可参见Spring 源码中  
  247. org.springframework.web.servlet.view.InternalResourceView.  
  248. exposeModelAsRequestAttributes方法的实现代码），也就是说Spring  
  249. MVC 中，结果数据对象默认都是requestScope。因此，在Spring MVC 中，  
  250. 以下寻址方法应慎用：  
  251. ${sessionScope.logininfo.username}  
  252. ${applicationScope.logininfo.username}  
  253. ? ${1＋2}  
  254. 结果为表达式计算结果，即整数值3。  
  255. ? ${i>1}  
  256. 如果变量值i>1的话，将返回bool类型true。与上例比较，可以发现EL会自  
  257. 动根据表达式计算结果返回不同的数据类型。  
  258. 表达式的写法与java代码中的表达式编写方式大致相同。  
  259. IF / CHOOSE:  
  260. <c:if test="${var.index % 2 == 0}">  
  261. *  
  262. </c:if>  
  263. 判定条件一般为一个EL表达式。  
  264. <c:if>并没有提供else子句，使用的时候可能有些不便，此时我们可以通过<c:choose>  
  265. tag来达到类似的目的：  
  266. <c:choose>  
  267. <c:when test="${var.index % 2 == 0}">  
  268. *  
  269. </c:when>  
  270. <c:otherwise>  
  271. !  
  272. </c:otherwise>  
  273. </c:choose>  
  274. 类似Java 中的switch 语句，<c:choose>提供了复杂判定条件下的简化处理手法。其  
  275. 中<c:when>子句类似case子句，可以出现多次。上面的代码，在奇数行时输出“*”号，  
  276. 而偶数行时输出“!”。

  301. 7、<c:otherwise>标签  
  302.      例：  
  303. 如果user.wealthy值true，则显示user.wealthy is true.  
  304. <c:choose>  
  305. <c:when test="">  
  306. user.generous is true.  
  307. </c:when>  
  308. <c:when test="">  
  309. user.stingy is true.  
  310. </c:when>  
  311. <c:otherwise>  
  312. user.generous and user.stingy are false.  
  313. </c:otherwise>  
  314. </c:choose>  
  315. 说明：只有当条件user.generous返回值是true时，才显示user.generous is true.  
  316. 只有当条件user.stingy返回值是true时，才显示user.stingy is true.  
  317. 其它所有的情况（即user.generous和user.stingy的值都不为true）全部显示user.generous and  
  318. user.stingy are false.  
  319. 由于JSTL没有形如if (){…} else {…}的条件语句，所以这种形式的语句只能用<c:choose>、  
  320. <c:when>和<c:otherwise>标签共同来完成了。  
  321. 8、c:forTokens>标签  
  322. 说明：

  325. begin 开始条件 否 0  
  326. end 结束条件 否 集合中的最后一个项目

  329. varStatus 显示循环状态的变量 否 无

  335. token是循环到当前分割到的字符串。
