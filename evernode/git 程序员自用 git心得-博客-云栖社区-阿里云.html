git 程序员自用 git心得

yexx 2016-11-21 17:51:00 浏览65 评论0

程序员 模块 测试 ADD git fetch Commit

摘要： 先看看本地有哪几个分支 git branch 一般都是在自己的分支上开发，然后合并到develop，所以可以看到我是在自己的分支上 然后你自己模块做完了，你需要提交到远程分支，你就先add一下，然后commit一下。 git add -A git commit -m 'test' 再切换到develop分支 git checkout develop

先看看本地有哪几个分支

git branch

一般都是在自己的分支上开发，然后合并到develop，所以可以看到我是在自己的分支上

然后你自己模块做完了，你需要提交到远程分支，你就先add一下，然后commit一下。

git add -A
git commit -m 'test'

再切换到develop分支

git checkout develop

因为要fetch了嘛，所以先看看我们的远程分支。

远程分支没有问题

git fetch origin develop

fetch之后就merge

git merge origin develop : develop

如果这个时候出现冲突，说明两个人同时改了一个文件，我们就解决，本地修改看是用远程的还是本地的。好了之后我们就可以push上去了。

好了这个时候我们的git就基本操作完了。master是不要动的，等大家功能都做完之后，我们测试没问题了再提交到master分支上。测试如果还有修改，我们也可以为测试新建一个分支。

Git操作之克隆某一个特定的远程分支

git clone -b <branch name> [remote repository address]
